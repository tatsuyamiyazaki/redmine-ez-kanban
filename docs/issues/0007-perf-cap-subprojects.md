# 0007 件数上限 + 超過バナー + サブプロジェクトトグル

Type: AFK

## What to build

大量チケット時にサーバー/DOM を保護しつつ、対象範囲を制御できるようにする。描画は上限で打ち切るが WIP 件数は総数を維持する（0003 の方針を保つ）。

- 描画カードの**グローバル件数上限**（既定 500、管理者が調整可）。上限内のカードのみ描画し、列ごとページングは行わない。
- 上限超過時に「結果が多すぎます。フィルタ/スコープで絞ってください」のバナーを表示。
- WIP 件数は上限に関係なく正確な総数のまま（0003 の GROUP BY COUNT を流用）。打ち切り時でも滞留検知が機能する。
- サブプロジェクトのチケットを含める include トグル。既定オフ（当該プロジェクトのみ）、状態は URL に保持（範囲A）。

## Acceptance criteria

- [ ] カード数が上限を超えると上限内のみ描画される
- [ ] 超過時にバナーが表示される
- [ ] 上限超過時も各列の WIP 件数は総数を示す
- [ ] サブプロジェクト include トグルがあり、既定オフ
- [ ] トグル状態が URL に保持され、リロードで復元される
- [ ] 上限・バナー・サブプロジェクト範囲のテストが通る
