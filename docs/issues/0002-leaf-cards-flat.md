# 0002 リーフカード（フラット表示）

Type: AFK

## What to build

ボードの核となるカード集合決定ロジックを end-to-end で通す。コントローラが既定で全ステータス（open+closed）の `IssueQuery` を構築し、その結果から**リーフ（子を持たないチケット）×フィルタ合致**の積（ADR-0002）でカード対象を決め、列分けはまだ行わず1枚のフラットなリストとして描画する。

- 既定クエリはステータス無制限（ADR-0005。Redmine 通常の open-only 既定を上書き）。
- リーフ判定は Redmine の nested set（`rgt = lft + 1`）でツリー全体に対して行う。フィルタとは独立したグローバル属性。
- チケット可視性は `Issue.visible` で必ず担保（権限のないチケットは出さない）。
- 各カードに ID・件名・担当者（アバター）・優先度・期日・トラッカーを表示（R8）。担当者は Redmine の `avatar` ヘルパーを使用。
- 閲覧専用: カードに編集・D&D 手段を一切設けない（R5）。カードクリックは Redmine のチケット詳細へ遷移。

## Acceptance criteria

- [ ] 子を持つ親チケットはカード化されない（A=非表示）
- [ ] フィルタ合致のリーフは、親がフィルタ非合致でも表示される（B=表示）
- [ ] フィルタ非合致のリーフは表示されない（C=非表示）
- [ ] closed なリーフも既定で表示される（既定全ステータス）
- [ ] 閲覧権限の無いチケットはカードに出ない
- [ ] カードに ID/件名/担当アバター/優先度/期日/トラッカーが出る
- [ ] リーフ判定・カード規則のユニットテスト（A/B/C シナリオ、子追加で消える R4 の挙動）が通る
