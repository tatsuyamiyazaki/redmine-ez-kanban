# 0003 列レイアウト・配置・WIP件数・列内ソート

Type: AFK

## What to build

フラットなカード集合を、ステータスをグループ化した固定列に配置して本来のカンバンにする。この時点では管理UIはまだ無く、`Setting` から読む**組込み既定設定**（未設定時のフォールバック）で動かす。配置・件数・ソートを end-to-end で通す。

- 列は `Setting` にシリアライズした単一グローバル設定から読む（ADR-0004）。未設定時の既定: 非クローズ各ステータスをステータス position 順に列化 + 1つの「完了」列（is_done）+ 末尾「未分類」列。
- 配置 precedence（ADR-0004）: 明示マッピング > `is_done` 列（未マッピングの is_closed） > 暗黙の末尾「未分類」列。マッピング先ステータスが削除/不明な場合は黙って未分類へフォールバック。
- 各列に WIP 件数を表示。件数は描画上限に関係なく**正確な総数**を、フィルタ＋スコープ条件に対する列ごと GROUP BY COUNT で算出（R10-1）。
- 列内ソート: クエリにソート指定があればそれに従う。無指定なら優先度降順 → 期日昇順（R9）。

## Acceptance criteria

- [ ] カードが既定設定の列に precedence 通り配置される
- [ ] is_closed の未マッピングステータスが「完了」列に入る
- [ ] 未マッピングの非クローズステータスが「未分類」列に入る
- [ ] 明示マッピングが is_closed→完了 のフォールバックより優先される
- [ ] 各列に正確な総件数が出る（描画上限の有無に依存しない）
- [ ] 列内がクエリ順 / 既定（優先度↓→期日↑）で並ぶ
- [ ] 配置 precedence・ソート・件数算出のユニットテストが通る
