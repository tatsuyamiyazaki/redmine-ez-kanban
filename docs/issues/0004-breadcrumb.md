# 0004 パンくず（親階層の文脈表示）

Type: AFK

## What to build

各カードに、ルートから直近の親までの祖先パスをパンくずとして表示し、リーフを階層をたどらず文脈付きで見渡せるようにする（R2）。0003 と並行可能。

- コントローラ/モデルで各リーフの祖先パス（root → 直近親）を nested set から算出。N+1 を避け、対象リーフの祖先をまとめて取得する。
- ビューはフルパスを描画しつつ、表示幅を超える場合は中間階層を省略し「ルート … 直近の親」形に短縮（R2-3）。完全パスは `title`/data 属性で持たせ、ホバー/ツールチップで全長表示（R2-4）。
- 親を持たない単独チケットのカードにはパンくずを表示しない（R2-5）。
- スイムレーンは設けず、あくまでフラット配置のままカード上の文脈表示とする（R2-1）。

## Acceptance criteria

- [ ] 親を持つリーフのカードに root→直近親のパンくずが出る
- [ ] 単独チケットのカードにはパンくずが出ない
- [ ] 幅超過時に中間が省略され「ルート … 直近親」になる
- [ ] 省略時、ホバー/ツールチップで完全パスが見える
- [ ] 祖先取得が対象カード数に対して定数本のクエリで済む（N+1 が無い）
- [ ] パンくず算出・省略ロジックのテストが通る
