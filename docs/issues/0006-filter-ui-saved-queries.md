# 0006 フィルタUI + 保存クエリ選択

Type: AFK

## What to build

Redmine 標準のフィルタ機構をボード上で操作できるようにし、保存クエリ・カスタムフィールドを含む絞り込みを利用可能にする（R7）。0002/0003 で内部的に構築していた `IssueQuery` に、ユーザー操作の UI を被せる。

- Redmine 標準のフィルタ partial をボードに埋め込み、既定では折りたたんで簡潔表示（R7-5）。
- 保存クエリ（プロジェクト既定クエリ含む）を選択できるセレクタを提供（R7-2）。
- フィルタ/選択クエリの状態を URL パラメータに保持し、共有・リロードで復元できる。
- カスタムフィールド条件など標準フィルタ条件がそのまま効く。可視性は引き続き `Issue.visible`/IssueQuery が担保（R7-3）。
- 既定（クエリ未選択）時はステータス無制限（ADR-0005）を維持。

## Acceptance criteria

- [ ] フィルタを変更するとボードのカード集合が更新される
- [ ] 保存クエリを選ぶとその条件でボードが描画される
- [ ] カスタムフィールド条件で絞り込める
- [ ] フィルタ/クエリ状態が URL に載り、共有・リロードで復元される
- [ ] フィルタUIは既定で折りたたまれている
- [ ] 既定（未選択）でステータス全件のままになっている
- [ ] フィルタ→クエリ構築のテストが通る
