# 0009 表示鮮度（手動更新 + 任意の定期ポーリング）

Type: AFK

## What to build

閲覧専用ボードの鮮度を、サーバー負荷を抑えつつ保てるようにする（非機能B）。コストのかかる挙動はオプトインにする（WIP 強調と同じ思想）。

- 常設の更新ボタンで、現在のフィルタ/スコープ/URL 状態を保ったままボード partial を再取得・差し替え（AJAX）。
- 任意の定期ポーリング。既定オフ。ユーザー/管理者が間隔（例 30/60/120 秒）を選んだ時のみ作動。
- ポーリング/手動更新は現在の全状態（フィルタ・スコープ・サブプロジェクト・上限）を維持して再描画。
- 再描画は読み取り専用で、ユーザーのスクロール位置等を不必要に壊さないこと。

## Acceptance criteria

- [ ] 更新ボタンで状態を保ったままボードが再描画される
- [ ] ポーリング間隔を選べ、既定はオフ
- [ ] 選択間隔でボード partial が自動差し替えされる
- [ ] 再描画後もフィルタ/スコープ/上限などの状態が維持される
- [ ] 更新経路がチケットへ一切書き込まない（R5 不変）
- [ ] 部分差し替えエンドポイントのテストが通る
